## Контекст

Рейтлимитер ограничивает частоту операций: например, не больше N вызовов API за секунду или за минуту. 
Он нужен, чтобы защитить сервис от перегрузки и злоупотреблений.

## Задача

Реализуйте класс `RateLimiter` в пакете `hse.java.lectures.lesson7.limiter`.

### API

- В конструктор передаются:
  - **единицу времени** — `java.time.temporal.ChronoUnit`: только `ChronoUnit.SECONDS` или `ChronoUnit.MINUTES` (скользящее окно длительностью 1 секунда или 1 минута соответственно; при другой единице — `IllegalArgumentException`);
  - **целое число** — максимально допустимое количество вызовов `check()` за это окно (строго положительное число; при неверных аргументах можно бросать `IllegalArgumentException`).
- Метод **`boolean check()`**:
  - учитывает текущий момент времени и историю вызовов в скользящем окне (последняя секунда или последняя минута);
  - возвращает **`true`**, если в окне на момент вызова **ещё есть место**: успешных попыток до этой **строго меньше**, чем переданный в конструктор лимит; при **`true`** текущий вызов **считается** в этом окне;
  - возвращает **`false`**, если лимит **уже достигнут** (успешных попыток уже не меньше, чем лимит); при **`false`** окно **не должно** вести себя так, будто сверхлимитный вызов «занял слот» у будущих успешных попыток.

Иными словами: за последнюю секунду/минуту не может быть **больше** успешных (`true`) вызовов, чем указано в конструкторе.

Учтите, что `check()` могут вызывать **несколько потоков одновременно** — реализация должна быть **потокобезопасной**.
